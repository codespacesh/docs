title: Team overview
description: Understanding roles and team membership
---

Each Codespace instance has its own team. The person who creates an instance is the **owner**, and they can invite others as **members**.

## Roles

| Permission | Owner | Member |
|------------|-------|--------|
| Create workspaces | Yes | Yes |
| Use workspaces | Yes | Yes |
| View instance details | Yes | Yes |
| Invite team members | Yes | No |
| Remove team members | Yes | No |
| Configure settings | Yes | No |
| Upgrade Coder version | Yes | No |
| Delete the instance | Yes | No |

## How membership works

- Membership is **per-instance** — a user can be a member of multiple instances
- Each instance has exactly **one owner** (the creator)
- Members are added via **email invitation**
- Invitations expire after a set period if not accepted

## Team management

Owners manage their team from the instance **Settings** page:

- [Invite members](/team/invite-members) — send email invitations
- [Manage members](/team/manage-members) — view, remove members, and revoke pending invitations

Members who receive an invitation follow the [accept invitation](/team/accept-invitation) flow to join.
