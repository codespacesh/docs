title: Subscription and usage
description: Manage your plan, view resource usage, and track overages
---

Each instance has a subscription plan that determines resource limits and features. View your subscription and usage from the instance **Settings** page.

## Your plan

The Settings page shows your current plan details:

- **Plan type** — Solo, Team, or Custom
- **Billing period** — start and end dates
- **Subscription status** — active, past due, etc.

See [Subscription plans](/reference/subscription-plans) for a full comparison.

## Resource usage

Usage meters show your consumption for the current billing period:

| Resource | Description |
|----------|-------------|
| **CPU hours** | Total compute time across all workspaces |
| **Memory GB-hours** | RAM usage across all workspaces |
| **Storage GB** | Persistent disk usage |

Each meter shows:

- Current usage vs. included amount
- A progress bar with percentage
- An **Overage** badge (red) if you've exceeded your included amount

<Warning>
  Usage that exceeds your plan's included amount is billed as overage. Monitor your usage to avoid unexpected charges.
</Warning>

## Workspace limits

Your plan also includes workspace limits:

- **Per-user limit** — maximum workspaces each member can create
- **Total limit** — maximum workspaces across the entire instance

## Upgrading your plan

The Settings page includes a plan comparison table showing what each tier includes. To upgrade:

- **Solo → Team** — contact support or use the upgrade flow
- **Team → Custom** — contact sales for custom pricing

<Tip>
  If you're consistently hitting overages, upgrading to the next plan tier is usually more cost-effective.
</Tip>
