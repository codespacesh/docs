Resources transition through several states during their lifecycle.

## Status reference

| Status | Badge color | Description |
|--------|-------------|-------------|
| **Pending** | Gray | Resource created, waiting for provisioning to begin |
| **Provisioning** | Yellow | Infrastructure is being set up (namespace, database, Coder, TLS, DNS) |
| **Running** | Green | Resource is healthy and accessible at `subdomain.codespace.sh` |
| **Failed** | Red | Provisioning encountered an error вҖ” can be retried |
| **Suspended** | Orange | Resource suspended due to billing (trial expired or subscription cancelled). Data is preserved but the service is inaccessible. Resume by activating a subscription. |
| **Deleting** | Yellow | Infrastructure teardown is in progress |
| **Deleted** | Gray | All infrastructure has been destroyed |

## What triggers transitions

| Transition | Trigger |
|-----------|---------|
| Pending вҶ’ Provisioning | Automatic вҖ” begins when resource is created |
| Provisioning вҶ’ Running | All infrastructure provisioned successfully |
| Provisioning вҶ’ Failed | A provisioning step encountered an error |
| Failed вҶ’ Provisioning | User clicks **Retry Provisioning** |
| Running вҶ’ Provisioning | User clicks **Upgrade** or **Force Reprovision** |
| Running вҶ’ Suspended | Trial expired or subscription cancelled (automatic) |
| Suspended вҶ’ Running | Subscription activated or resumed (automatic via reconciler) |
| Running вҶ’ Deleting | User clicks **Delete** and confirms |
| Deleting вҶ’ Deleted | All resources destroyed |

## Dashboard behavior

- The dashboard polls every 5 seconds for status updates
- Status badges update automatically вҖ” no page refresh needed
- The Events tab shows detailed progress during provisioning
